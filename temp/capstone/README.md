## Quick start

```bash
# 1 – Install dependencies (inside your venv)
pip install -r requirements.txt

# 2 – Generate the 120,000-row dataset
python generate_data.py

# 3 – Run the SIMPLE pipeline (baseline — no design patterns)
python simple_pipeline.py

# 4 – Run the ADVANCED pipeline (dev environment)
python main.py

# 5 – Switch to prod config
PIPELINE_ENV=prod python main.py

# 6 – Run tests
pytest tests/ -v
```

## Pipeline flow (advanced)

```
CSV file (120K rows)
    │
    ▼  [Module 19] asyncio — fetch category metadata for 5 categories concurrently
    │
    ▼  [Module 19] ProcessPoolExecutor — parallel pre-validation across 4 cores
    │
    ▼  CSVReader.read()               ← generator, row-by-row  [Module 16]
    │
    ▼  CleaningTransformer.transform() ← generator → drops ~3 % bad records [Module 15, 16, 20]
    │
    ▼  EnrichmentTransformer.transform() ← computes financials, injects async metadata [Module 15, 16]
    │
    ├─► CSVWriter          → output/dev/cleaned_sales.csv     [Module 15]
    │
    ▼  AggregationTransformer.transform() ← groups by region/category/year/quarter [Module 15]
    │
    ├─► JSONWriter          → output/dev/aggregated_summary.json
    └─► SummaryWriter       → output/dev/pipeline_report.md
```

## Key comparison: simple vs advanced

| Aspect | `simple_pipeline.py` | `main.py` |
|--------|---------------------|-----------|
| Memory | Loads full dataset twice | Streams row-by-row |
| Config | Hard-coded paths | YAML per environment |
| Logging | `print()` only | Rotating file + console |
| Errors | Crashes on first bad row | Validates, skips, counts |
| Retry | None | Exponential back-off |
| Parallelism | None | ProcessPoolExecutor + asyncio |
| Testable | No | 50 + pytest tests |
| Reusable | No | ABCs & decorators composable |

## Sample outputs

After running `python main.py` you will find in `output/dev/`:

* **`cleaned_sales.csv`** — ~116 K rows with added columns:
  `order_year`, `order_quarter`, `gross_revenue`, `net_revenue`,
  `gross_margin`, `margin_pct`, `category_tier`, `is_high_value`

* **`aggregated_summary.json`** — one JSON object per
  `(region, category, year, quarter)` bucket with totals and KPIs

* **`pipeline_report.md`** — Markdown report with overall KPIs,
  revenue by region/category and top-10 margin buckets

* **`logs/pipeline_dev.log`** — full DEBUG trace of every step
