# OOP Reference — Python (Simple)

This short reference lists core object-oriented concepts used in Python, with one-line explanations.

- class – Define a blueprint for objects.
- object – Base type that most Python classes inherit from implicitly.
- __init__ – Constructor method that initializes new instances.
- self – Conventional name for the instance parameter in methods; not a keyword.
- inheritance – A class can extend another to reuse and override behavior.
- super – Call a method from a parent class.
- method overriding – Replace a parent method implementation in a subclass.
- encapsulation – Keep data and methods together; use naming conventions for privacy.
- composition – Build complex behavior by containing other objects.
- polymorphism – Different types can be used interchangeably if they share behavior.
- abstract base class (ABC) – Define a contract with `abstractmethod` that subclasses must implement.
- abstractmethod – Mark a method in an ABC that must be implemented by subclasses.
- interface (informal) – A set of expected methods/behaviors; Python relies on duck typing.
- duck typing – Use objects based on available methods rather than explicit types.
- property – Define getter/setter behavior for attribute access using `@property`.
- del – Remove attributes or names from scope.

---

Notes: Keep implementations simple; prefer composition over complex inheritance when possible.
